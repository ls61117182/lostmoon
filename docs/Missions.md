## 任务 1：村庄

> 说明书 P26。对应数据：`assets/resources/missions/mission_01.json`。

### 目标

- 摧毁 **2 辆 IV 号坦克**；
- 随后从 **红色箭头方向**（地图顶边，向北）移动离开地图。

### 初始设置

- **谢尔曼**：放置在地图 **右下角的黑色箭头格**（公路末端），朝向箭头所指方向（西北，驶入地图）。
  - 数据中固定为 `col=6, row=9, facing=4 (NW)`；pointy-top 六角无正北，
    NW 是最接近照片里黑色箭头角度的方向。
- **2 辆 IV 号坦克**：在地图上的 **5 个黑色数字格**（`eid=2,3,4,5,6`）中随机
  选择 **2 个不同位置** 放置，每辆朝向其所在编号在六边形上的那条格边。
  - 当前 JSON 采用固定布置：`eid=2`（`col=4,row=2`，右上）放一辆朝 SW(2)、
    `eid=3`（`col=6,row=3`，右中）放一辆朝 W(3)。若需严格遵循规则书的
    随机性，可在载入时对 `tiles` 中所有带 `eid` 的格子随机抽 2。
- **援军**：红色数字格 `rid=1~6` 共 6 处，分布在公路沿线与左下角，作为未来
  援军事件的生成位预留。
- **步兵 / 机枪测试**：规则书任务 1 不含步兵。机枪掷骰动画在后续任务或
  测试场景中验证；本任务保持"只打坦克"的纯净对局。

### 胜负条件（代码实现）

- 使用 `objective.type = "destroy_kind"`、`kind = "panzer4"`，即击毁全部
  IV 号坦克即判胜；失败条件沿用主循环中谢尔曼被摧毁的逻辑。
- **待办**：规则书的"击毁 + 撤离"复合目标暂未在引擎实现；后续可扩展为
  `destroy_kind` + `exit_from_edge` 的串联条件，`exitDirection` 设为
  `4 (NW)` 或 `5 (NE)` 表示从顶边离场。

### 地图说明（7 列 × 10 行）

坐标系：`tiles[row][col]`，odd-r 偏移。`row=0` 为顶边（红色箭头出口，位于
`col=2` 上方），`row=9, col=6` 为黑色箭头谢尔曼起始格。地形字符见
`MissionLoader.ts`：

| 字符 | 含义                              |
| ---- | --------------------------------- |
| `r`  | 公路（移动 +1 骰）                |
| `f`  | 田地                              |
| `m`  | 泥地 / 碎石（移动 −1 骰）         |
| `F`  | 林地（坦克不可入，阻挡视线）      |
| `b`  | 建筑（坦克不可入，阻挡视线，+1） |
| `w`  | 水域（任何单位不可入）            |

树篱（bocage）使用 `h` 字段，一个 6 位 `0/1` 字符串，按 `Direction` 顺序
`[E, SE, SW, W, NW, NE]` 指示六条格边上是否有树篱。

示意布局（字母 = 地形；`🔴n` = 援军红数字；`⚫n` = IV 号黑数字；`↖S` = 谢尔曼起始位朝 NW；`‖` 表示该格带完整 bocage 围合；`•` 泥地村庄残骸；`F` 林地）：

```
         col0   col1   col2   col3   col4   col5   col6
row0:    f      f      r🔴1  f      F      f      f         ← 顶边红色箭头↑（出口位于 col 2）
row1:    f⚫6   ‖f‖    r      f      f      f      f
row2:    f      f      r      f      f⚫2   f      f
row3:    •m     ‖f‖    r      f🔴4   •m     F      f⚫3
row4:    f      •m     r🔴3  •m     f      f      f         ← 中央村庄（公路夹在残骸中）
row5:    f⚫5   F      f      r      f      f      f
row6:    f      f      f      r      f      f🔴5   f
row7:    f🔴2   f      f      •m     r      f      f
row8:    f⚫4   f      f      •m     r      f      f
row9:    f      f      f      f🔴6   f      r      r↖S      ← 底边黑色箭头
```

- **公路**：自顶边 `col 2` 垂直下行 `(2,0)→(2,1)→(2,2)→(2,3)→(2,4)` 穿过村庄，
  随后向东南折 `(3,5)→(3,6)→(4,7)→(4,8)→(5,9)→(6,9)`，最终连到右下角
  谢尔曼起始格 `(6,9)`。整条单线公路共 11 格。
- **村庄（中段）**：`(0,3)/(4,3)` 与 `(1,4)/(3,4)` 四片泥地构成公路两侧的
  残骸村庄，公路从 `(2,3)(2,4)` 两格穿村而过（对应照片里 ③红 所在位置）。
- **村庄（南段）**：`(3,7)/(3,8)` 两格泥地是地图南部的残骸堆，
  对应照片里 ⑥红 周围的木箱 / 坦克残骸图标。
- **Bocage（树篱）**：除公路/泥地外几乎每块田地都用 `h` 标注了数条树篱边；
  `(1,1)` 与 `(1,3)` 用 `h="111111"` 围合成完整的小型 bocage 田块。
- **林地 `F`**：3 格 —— `(4,0)` 顶中、`(5,3)` 中右、`(1,5)` 中左，用于阻挡两翼视线。
- **黑色数字格 `eid`**：5 处，对应照片中 ⑥②③⑤④ 五个黑圈：
  `(0,1)⑥`、`(4,2)②`、`(6,3)③`、`(0,5)⑤`、`(0,8)④`。
- **红色数字格 `rid`**：6 处，对应照片中 ①④③⑤②⑥ 六个红圈：
  `(2,0)①`、`(3,3)④`、`(2,4)③`、`(5,6)⑤`、`(0,7)②`、`(3,9)⑥`。
